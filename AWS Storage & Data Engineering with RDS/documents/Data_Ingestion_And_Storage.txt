PART 3: DATA INGESTION & STORAGE (Python + Boto3)

Objective:
The objective of this phase is to ingest raw data from the local system into cloud storage and manage it using Python scripts. This ensures the dataset is stored in a scalable and organized data lake architecture for further processing and analytics.

Services and Tools Used:
- Amazon S3 for cloud storage
- Python for scripting
- Boto3 (AWS SDK for Python) to interact with AWS services

------------------------------------------------------------

Step 1: Upload Raw CSV Dataset to S3

Dataset Used:
cleaned_ecommerce_data.csv

Task Performed:
The dataset was uploaded from the local system to the S3 bucket created in Part 2.

S3 Bucket Name:
nafisa-s3-project-2026

Location in Bucket:
/raw/cleaned_ecommerce_data.csv

Purpose:
The raw folder acts as the landing zone for original datasets before they are processed or transformed.

------------------------------------------------------------

Step 2: Organize Data Using S3 Folder Structure

A structured folder hierarchy was created inside the S3 bucket to organize the data efficiently.

Bucket Structure:

nafisa-s3-project-2026/
    raw/
    processed/
    logs/

Folder Purpose:

raw/
Stores original datasets uploaded from local systems.

processed/
Stores datasets after transformation and processing.

logs/
Stores execution logs generated by scripts or pipelines.

This structure follows common data lake practices used in cloud data engineering workflows.

------------------------------------------------------------

Step 3: List Files in the S3 Bucket Using Python

Script Used:
list_s3_files.py

Task Performed:
This script connects to the S3 bucket using Boto3 and retrieves the list of all objects stored in the bucket.

Outcome:
The script confirmed that the dataset cleaned_ecommerce_data.csv was successfully uploaded to the raw folder.

------------------------------------------------------------

Step 4: Download Files from S3 Using Python

Script Used:

Source Path in S3:
/raw/cleaned_ecommerce_data.csv

Result:
The dataset was successfully downloaded from S3 to the local project directory.

------------------------------------------------------------

Step 5: Archiving Older Data Using Amazon Glacier

To optimize storage costs and manage long-term data retention, older or infrequently accessed data can be moved to archival storage.

Service Used:
Amazon S3 Glacier

Archival Strategy:
Lifecycle policies can be configured in the S3 bucket to automatically move data from standard storage to Glacier after a specific time period.

Example Strategy:
After 30 days → Move data to Standard-Infrequent Access
After 90 days → Move data to Amazon S3 Glacier
After 365 days → Delete or archive permanently

Benefits:
- Lower storage cost
- Secure long-term data retention
- Automated lifecycle management
